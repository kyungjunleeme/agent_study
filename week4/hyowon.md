# Part2. 생성형 AI 기반 에이전트 설계 및 구현
# CH4. 에이전트의 반성과 자기 성찰

## 1. 에이전트에서의 반성의 중요성
- 향상된 의사결정
- 적응
- 윤리적 고려
- 인간-컴퓨터 상호작용
- 지능형 에이전트의 자기 성찰

## 2. 반성 기능 구현하기
다음은 반성 기능을 구현하기 위한 기법들이다.
### 전통적인 추론
사전에 정의된 규칙, 알고리즘, 데이터로부터 학습된 패턴에 따라 문제 해결하는 과정
### 메타 추론
에이전트가 자신의 추론 활동을 모니터링하고 제어
조정은 성능 모니터링, 자원 할당 등
- 성능 모니터링
	- 여행 에이전트 기준으로 성공률 모니터링, 사용자 만족도 추적
	- 추적 가능 지표: 사용자 평점/리뷰, 추천 수용률, 불만 및 취소율, 사용자 참여 지표, 인구 통계 기반 통찰
- 리소스 할당
	- 복잡하거나 중요도 높은 여행 계획은 더 많은 리소스 할당
	- 일상적이거나 간단한 요청은 간소화된 휴리스틱이나 사전에 정의된 규칙에 의존
	- 다양한 알고리즘: 강화 학습, 다중 슬롯 머신, 베이지안 최적화, 동적 프로그래밍, 휴리스틱 기반, 게임 이론적 접근
### 자기 설명
에이전트가 수행한 추론 과정을 언어화해 의사결정에 대한 설명을 생성
활용
- 투명성 증진: 인간의 이해 돕기
- 학습 촉진: 에이전트가 개선을 위해
- 사용자 참여 및 협업: 사용자와의 양방향 대화 가능. 개인화 가능.
### 자기 모델링(self-modeling)
자신의 목표, 신념, 지식에 대한 내부 표현을 유지하는것.
modeling? 에이전트의 초기 환경과 상태를 의미

```
 [지식] - [에이전트1] - [목표]
              |
 [지식] - [에이전트2] - [목표]
* 자기 모델링을 위한 개별 및 공유 내부 상태
```

- 목표 관리
	- 목표: 사용자 선호와 제약 조건에 따라 여행 경험을 최적화
	- 목표는 새로운 상황이나 정보에 따라 에이전트가 재고하고 조정한다
- 지식 업데이트
	- 새로운 경험과 통찰을 바탕으로 에이전트의 지식 베이스를 자동으로 업데이트
