
# Building Agentic AI Systems

## Chapter 2 : Principles of Agentic Systems
2장에서는 Agent의 특징에 대해서 설명하고, Agent 특성에 따른 아키텍쳐의 종류를 나열/설명하고, 
Multi Agent System에 대한 간략한 설명을 덧붙인다.

## Agent가 가져야 할 특징들
세 가지로 설명하는데. Self-Governance, Agency, Automony라고 하는데.
독립적으로 선택할 수 있어야만 Agent라고 할 수 있다는 말처럼 들린다. 
다양한 설명들은 그냥 별로 의미가 없어 보인다. Agency애서 의도성(Intentionality)가 나오는데. 별로 와 닿지는 않았다.
예제 Trip Advisor Agent도 의도를 가졌다고 하긴 힘든 것 같다. 

## 아키텍쳐 종류
* Deliberative: 환경에 대한 이해를 Knowledge base로 저장하고 사고하는 기능을 주요 요소로 가지고 있다.
* Reactive: 환경에 대한 판단과 행동이 연계되여 작동. 속도가 빠르고 분산처리가 용이하다.
* 혼합형

## Multi Agent System
다른 설명들은 Agent 특징에서 파생된 것으로 보이고, Interaction 정도만 새로운 것이로 보인다.
정보를 교환하는, 넓은 의미에서 협업하는, 협상하는 프로토콜이 있어야 한다고 하는데. 
협상(negotiation)은 어떻게 실제로 정의될지 궁금하기는 한다.
